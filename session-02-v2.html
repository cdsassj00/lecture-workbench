Fuse.js 기반 정적 챗봇

강사 사이트 cdsassj00.github.io/2026-proposals-rss의 챗봇은 LLM이 아니라 클라이언트 fuzzy search 라이브러리 Fuse.js로 구현된 정적 검색기다.

구성 요소
DATA
JSON 형태로 정적 임베딩된 Q·A 페어 + 제안서 본문
ENGINE
Fuse.js · Bitap 알고리즘 기반 fuzzy string match
SCORING
키워드 가중치 + 거리 기반 score (0~1, 낮을수록 일치)
UI
바닐라 JS · 외부 LLM API 호출 없음
LLM 챗봇과의 차이
  • 생성 능력 없음 — 미리 들어간 답변 풀에서 가장 가까운 항목을 고를 뿐
  • 오프라인 동작 — 인터넷 없이 정적 호스팅에서 그대로 작동
  • 비용 0원 — API 호출 없음, 토큰 과금 없음
  • 한계 — 신규 질문·복합 질의 처리 불가

ChatGPT 실습 — 후보 목록

아래 후보 중 강사가 시간·수강생 분포에 맞춰 선택한다. 산출물은 구글드라이브 공용 폴더 또는 GitHub 레포에 업로드.

  1. SVG 도식 생성 — 부서 업무 흐름·조직도·데이터 구조 중 1개

  5. 첨부 파일 분석 — PDF·엑셀에서 표·수치·인용문 추출
  6. Custom GPT 구성 — 시스템 프롬프트·지식 첨부·동작 정의로 1개 챗봇

09 · GitHub + 노코드 바이브코딩

Lovable · Bolt · v0 · Replit

자연어 입력 → 작동하는 웹앱·UI 컴포넌트. GitHub 레포로 export 가능한 4개 도구를 비교.

Lovable
  • 강점 · 자연어 → 풀스택 (React + Tailwind + Vite)
  • 백엔드 · Supabase 연동이 표준 흐름
  • 출력 · GitHub 레포 직접 동기화 · Vercel 배포
  • 적합 · 부서 내부 도구 1차 프로토타입
Bolt (StackBlitz)
  • 강점 · 브라우저 안에서 풀스택 환경 실행 (WebContainer)
  • 스택 자유 · React · Vue · Svelte · Next · Astro 등
  • 출력 · GitHub 레포 export
  • 적합 · 단발성 데모·POC
v0 (Vercel)
  • 강점 · UI 컴포넌트 자연어 디자인 (shadcn/ui · Tailwind)
  • 출력 · React 컴포넌트 코드 · Next.js 페이지
  • 배포 · Vercel 원클릭
  • 적합 · 화면·컴포넌트 프로토타입
Replit
  • 강점 · 클라우드 IDE + Agent · 코드를 직접 보고 수정
  • 스택 자유 · 거의 모든 언어·프레임워크
  • 출력 · GitHub 연동 · Replit 호스팅
  • 적합 · 노코드 → 코드 전환 중간 단계

09 · 실습 시나리오 후보 (강사 선택)

노코드 4종 실습 — 후보 목록

  1. Lovable — 입력 폼 + 결과 표시 페이지 1개. Supabase 테이블 1개 연결.
  2. Lovable — 부서 안내 단일 페이지를 자연어로 생성, GitHub 푸쉬
  3. Bolt — 정적 HTML 한 페이지를 5분 안에 생성, 즉시 미리보기
  4. Bolt — Next.js·Astro 중 하나로 다국어 안내 페이지 1개
  5. v0 — 대시보드 한 화면을 React 컴포넌트로 생성, props 변경으로 변형
  6. v0 — 폼 컴포넌트 1개 생성 후 v0에서 직접 미세 조정
  7. Replit — Lovable 결과물을 import하고 1개 함수만 손으로 수정
  8. 4종 비교 — 동일 요구("부서 민원 입력 폼")를 4개 도구에 같이 입력하고 결과 비교
