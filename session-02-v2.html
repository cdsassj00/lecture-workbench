Fuse.js 기반 정적 챗봇

강사 사이트 cdsassj00.github.io/2026-proposals-rss의 챗봇은 LLM이 아니라 클라이언트 fuzzy search 라이브러리 Fuse.js로 구현된 정적 검색기.

구성 요소
DATA
JSON 형태로 정적 임베딩된 Q·A 페어 + 제안서 본문
ENGINE
Fuse.js · Bitap 알고리즘 기반 fuzzy string match
SCORING
키워드 가중치 + 거리 기반 score (0~1, 낮을수록 일치)
UI
바닐라 JS · 외부 LLM API 호출 없음
LLM 챗봇과의 차이
  • 생성 능력 없음 — 미리 들어간 답변 풀에서 가장 가까운 항목을 고를 뿐
  • 오프라인 동작 — 인터넷 없이 정적 호스팅에서 그대로 작동
  • 비용 0원 — API 호출 없음, 토큰 과금 없음
  • 한계 — 신규 질문·복합 질의 처리 불가

04 · ChatGPT

ChatGPT — 7개 고급 기능

모델 라인업은 시기별로 변동(GPT 시리즈·o-시리즈·mini). 라인업보다 고급 기능 7가지를 익힌다.

01 · PROJECTS
대화·파일·지침을
한 작업 공간에 묶기
올린 파일을 이후 모든 대화가 자동 참조. 부서 단위 배포 가능.
02 · CUSTOM GPTs
시스템 프롬프트 +
파일 + Actions 패키징
프롬프트의 패키징. 부서 특화 챗봇으로 동료에게 배포.
03 · CANVAS
옆 패널에서
문서·코드 편집
문단·코드 블록 단위 수정. 외부 연결 없음 = 보안 제약 덜함.
04 · AGENT MODE
멀티스텝 자동 실행 +
외부 도구 연결
Drive 검색 → 정리 → 보고서 순차 실행. 사람 승인 게이트 필수.
05 · o1 / o3 REASONING
"생각 시간"을 쓰는
추론 전용 모델
법령 해석·조건 충돌 점검 같은 다단계 판단에 강함. 비용·속도 ↑.
06 · MEMORY
사용자 정보·맥락을
대화 사이 자동 기억
부서·담당 업무 한 번 알려주면 이후 대화에서 반영.
07 · DEEP RESEARCH
수십 웹 소스 자동 순회 →
종합 보고서
정책 벤치마킹 초안용. 출처 인용 자동.

Grok · Z.AI · 중국 3종 + ZONE 분류

3사 외 대안 5종은 인식 수준으로만 훑는다. 데이터 위치별 ZONE A/B/C 분류 — 같은 기능이라도 ZONE이 갈린다.

대안 5종 — 접속 체험만
GROK
xAI · X 타임라인 직접 접근 · 실시간 여론 감지 (편향 큼, 보조용)
Z.AI (GLM)
Zhipu/智谱 · GLM 시리즈 · 웹 오케스트레이션 (중국 서버)
DEEPSEEK
추론 특화 R1 · 오픈 가중치 공개 (서비스는 중국 서버)
QWEN
알리바바 · 다국어 · 오픈 가중치 (서비스는 중국 서버)
MINIMAX
장문 컨텍스트 · Hailuo 영상 (중국 서버)
데이터 위치 분류 — ZONE A/B/C
ZONE A
완전 로컬 · Ollama·LM Studio. 외부 통신 0. 민감 문서 가능.
ZONE B
외부망 (미국 등 신뢰권) · ChatGPT·Claude·Gemini·Perplexity. 비식별화 후.
ZONE C
중국 서버 경유 · Z.AI·DeepSeek·Qwen·MiniMax·Kling. 공공 데이터 업로드 엄금.
예외
오픈 가중치(Qwen·DeepSeek)를 로컬로 끌어쓰면 ZONE A로 이동 가능
⚠   RULE — 중국 서버(ZONE C) 서비스에 공공 데이터·내부 보고서·개인정보 업로드 절대 금지. 일반 공개 텍스트로 UI·응답 속도·한국어 품질만 5분 체험.

적용 후보 발굴 — 토의 주제

기능과 적용 가능성에 집중. 강사가 진행 형식(개인·그룹·화이트보드)을 선택.

  1. 본 부서에서 반복도가 가장 높은 작업 1개 + 적용 가능한 LLM 기능 매칭

  4. 부서 자료의 ZONE 분류 — A/B/C 어디에 어떤 자료를 둘 것인가
  5. 3사(ChatGPT·Gemini·Claude) 중 본 부서 1차 도입 후보 + 근거
  6. Claude Skills로 묶어둘 만한 우리 부서 반복 절차 1~2개
