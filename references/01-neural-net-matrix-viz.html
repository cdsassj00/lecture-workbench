신경망 = 행렬 곱셈

5×5 픽셀이 가중치 행렬을 만나 어떻게 X·O·+·T로 분류되는가 — 모든 숫자를 보여주는 시뮬레이터

01 / 입력

5×5 픽셀, 25개의 숫자

픽셀을 클릭해 직접 그려보거나, 프리셋을 눌러 보세요. 켜진 픽셀은 1, 꺼진 픽셀은 0. 이 25개 숫자가 입력 벡터 x입니다.

PRESETS
02 / 레이어 1 · 패턴 검출

h = ReLU(x · W₁ + b)

6개의 뉴런이 각자 다른 패턴을 찾습니다. 각 뉴런마다 5×5 가중치 행렬 W가 있고, 같은 위치끼리 곱한 후(W ⊙ x) 25개를 다 더해서 합을 냅니다. 거기에 bias(-2)를 더하고 ReLU(음수면 0)를 씌우면 활성화 값 h가 나옵니다. 곱이 1인 칸은 진한 주황으로 표시 — "패턴이 매치된 곳"입니다.

03 / 레이어 2 · 점수 계산

score = h · W₂ + b₂

앞에서 만든 h(6차원)가 이번엔 입력입니다. 4개 클래스 각각이 자기 가중치(6개씩)와 h를 곱해서 점수를 냅니다. 양의 가중치는 "이 패턴이 있으면 좋다", 음의 가중치는 "이 패턴이 있으면 다른 클래스다".

04 / Softmax

점수 → 확률 → 예측

Softmax가 4개 점수를 모두 양수로 만들고 합이 1이 되도록 정규화합니다. 가장 높은 확률을 가진 게 예측.

P(classi) = exp(scorei) / Σ exp(scorej)
PREDICTION
·
05 / 트랜스포머 연결

Q, K, V는 무엇이고, 멀티헤드는 무엇인가

방금 우리가 본 "입력 × 행렬 + bias → 활성화" 구조는 트랜스포머 어텐션의 Q/K/V를 만드는 과정과 정확히 동일합니다. 다만 그 결과를 어디에 쓰는지가 다를 뿐입니다.

① Q는 뉴런 한 개가 아닙니다 — Q는 벡터입니다

"Q value가 뉴런 한 개냐"는 질문에 답하자면 아닙니다. Q는 벡터예요. 숫자 여러 개의 묶음.

우리 모델에서 뉴런 1개(예: h₁)는 출력으로 숫자 1개만 냅니다. 그런데 트랜스포머의 Q 벡터는 보통 64개 숫자입니다. 그러니까 비유하자면 Q 벡터 ≈ 우리 모델의 h 벡터예요. h가 [h₁, h₂, ..., h₆] 6개 묶음이듯, Q도 [q₀, q₁, ..., q₆₃] 64개 묶음.

한 줄 요약: 우리 모델의 h = ReLU(x · W₁ + b) 한 줄이, 트랜스포머에서는 Q = x · W_Q, K = x · W_K, V = x · W_V 세 줄로 나뉩니다. 곱셈 구조는 똑같고, 결과를 무엇에 쓰느냐가 다릅니다.

② Q, K, V는 각자 무슨 역할인가

토큰(단어) 하나가 들어오면 임베딩 벡터 x가 됩니다. 거기에 세 개의 가중치 행렬을 곱해서 세 벡터를 만듭니다.

Q = x · W_Q // "내가 무엇을 찾고 싶은지"
K = x · W_K // "내가 무엇을 가지고 있는지"
V = x · W_V // "실제로 전달할 정보"

그 다음 어텐션은 다른 토큰의 K와 내 Q를 dot product해서 "얼마나 관련 있는지" 점수를 매깁니다. 그 점수로 V들을 가중평균해서 출력을 냅니다. dot product도 결국 같은 위치끼리 곱해서 다 더하는 행렬 곱셈 — 우리가 위에서 본 그것과 동일한 연산.

③ 그러면 멀티헤드(multi-head)란?

위의 W_Q, W_K, W_V 한 세트를 "한 헤드"라고 부릅니다. 멀티헤드는 이걸 8개~16개 평행하게 두는 겁니다. 각 헤드가 다른 가중치를 학습하기 때문에, 어떤 헤드는 문법 관계, 어떤 헤드는 의미 유사성, 어떤 헤드는 위치 관계를 잡아냅니다.

우리 모델로 비유하면 — 우리 Layer 1은 6개 뉴런이 각자 다른 시각 패턴(↘, ↙, —, |, □, ⊤)을 찾습니다. 멀티헤드는 이런 "검출기 6개 묶음"을 8벌, 16벌 평행으로 두고 결과를 이어붙이는 셈입니다. 한 묶음은 모서리 검출, 한 묶음은 곡선 검출, 한 묶음은 색상 검출 — 이런 식.

그래서 우리 레이어 = 멀티헤드? 정확히는 아닙니다. 우리 Layer 1은 한 헤드의 W_Q 한 개에 가깝습니다. 멀티헤드가 되려면 같은 입력 x에 대해 W_Q를 8벌 평행하게 두고 각각 Q를 8개 만들어야 합니다.

④ 한눈에 비교

항목우리 모델 (이 시뮬레이터)트랜스포머 어텐션
입력 5×5 픽셀 → 25차원 벡터 x 토큰 임베딩 → 보통 512차원 벡터 x
가중치 행렬 W₁ (25 × 6) — 1개 W_Q, W_K, W_V (각 512 × 64) — 3개
한 번 곱셈으로 나오는 것 h_pre 6차원 → ReLU → h Q 64차원 / K 64차원 / V 64차원
"뉴런 1개"란? 출력 스칼라 1개 — h₁부터 h₆까지 6개 벡터의 한 칸 — Q[0], Q[1], ..., Q[63]
레이어 한 개의 의미 패턴 검출기 6개 묶음 한 헤드 = (W_Q + W_K + W_V) 묶음.

결과를 어디에 쓰나 다음 레이어 → Softmax → 분류 Q·K로 attention score → V를 가중평균 → 다음 레이어
활성화 함수 ReLU (음수면 0) Softmax (어텐션 score 정규화) + GELU (FFN)
최종 결론: "행렬 곱셈 + 활성화" 한 토막이 신경망의 기본 단위입니다. 분류 모델은 이걸 직선으로 쌓고, 트랜스포머는 같은 단위를 Q/K/V 세 갈래로 만든 뒤 토큰 간 관계 계산에 씁니다. 구조가 어려운 게 아니라, 같은 구조를 어떻게 조합하느냐가 어려운 겁니다.
