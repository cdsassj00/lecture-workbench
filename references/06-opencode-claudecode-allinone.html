오픈코드(OpenCode) · Claude Code — 슬래시 명령 / 툴 / 설정 방식 올인원 가이드 (일반 사용자용)

이 문서는 “AI 코딩 도구”를 처음 쓰는 사람도 이해할 수 있도록, 다음을 한 파일에 빠짐없이 모았습니다.

1) OpenCode의 슬래시 명령(/...)과 커스텀 명령 작성 방식 2) Claude Code의 슬래시 명령(/...)과 설정/확장(메모리, 권한, 훅, 스킬, 플러그인, MCP) 3) (참고) 이 채팅/에이전트 환경에서 실제로 호출 가능한 Tool/Skill 목록(인벤토리) 4) Cursor/Copilot/Aider/Windsurf 등과 비교했을 때 차별점(기술/설정 방식)

중요한 전제: 두 제품 모두 버전/플랫폼/요금제/환경에 따라 보이는 명령이 달라질 수 있습니다. 따라서 “지금 내 환경에서 진짜로 하나도 빠짐없이 보는 방법”은 OpenCode/Claude Code 모두에서 /를 입력해 명령 팔레트(목록)를 여는 것입니다.

1) 1분 용어 정리

  • 슬래시 명령(/...): 대화 중 /로 시작하는 내장 명령/워크플로우.
  • 커스텀 슬래시 명령: 사용자가 마크다운 파일로 직접 만드는 /my-command 같은 명령.
  • Tool(툴): AI가 실제로 실행하는 기능(파일 읽기/검색/수정, 터미널 명령 실행 등).
  • Skill(스킬): 특정 작업을 잘 하기 위한 “워크플로우 묶음”(문서 + 규칙 + 스크립트 + 참고자료).
  • Plugin(플러그인): 슬래시 명령/스킬/훅/에이전트를 묶어 배포하는 패키지.
  • MCP: 외부 시스템(깃헙, DB, 노션 등)을 AI에 연결하는 표준 프로토콜.

2) 다른 코딩툴(Cursor/Copilot/Aider 등)과의 ‘결정적 차이’

일반 사용자가 체감하는 차이만 핵심만 뽑으면:

1) 설정의 구조화(메모리/규칙 파일)

  • Claude Code: CLAUDE.md(계층형)로 팀 규칙/금지사항/작업순서를 지속 적용
  • OpenCode: AGENTS.md(프로젝트 컨텍스트) + .opencode/commands로 반복 작업을 명령으로 캡슐화

2) 슬래시 명령 생태계

  • “명령이 곧 워크플로우”가 되어, 반복 작업을 메뉴/명령으로 실행 가능

3) 확장 방식(스킬/플러그인/MCP/훅)

  • 단순 채팅이 아니라, 자동화/분업/외부도구 연결까지 포함한 플랫폼 형태

4) 권한/안전 장치가 1급 기능

  • 파일 수정/터미널 실행/외부 연결을 세밀하게 허용/차단

3) 공식 문서(정확도 기준점)

3.1 Claude Code Commands(명령 전체 레퍼런스)

  • Claude Code 명령 레퍼런스(공식): https://code.claude.com/docs/en/commands
  • 페이지에 “모든 명령 표”가 있으며, 플랫폼/요금제별로 일부만 보일 수 있다고 명시합니다.
  • 내 환경에서 빠짐없이 확인: 세션에서 / 입력 → 목록/검색

3.2 OpenCode Slash Commands(명령 전체 레퍼런스)

  • OpenCode 슬래시 명령(공식): https://www.opencode.asia/reference/slash-commands/
  • 내 환경에서 빠짐없이 확인: 세션에서 / 입력 → 목록

4) OpenCode — 슬래시 명령 “전체 목록”(공식 문서 기준)

아래는 OpenCode 공식 문서의 Quick Reference를 한국어로 정리한 것입니다.

4.1 핵심 명령(Quick Reference)

명령별칭단축키용도
/help-Ctrl+X H도움말
/new/clearCtrl+X N새 세션 시작(컨텍스트 리셋)
/undo-Ctrl+X U직전 메시지 되돌리기(※ Git 필요)
/redo-Ctrl+X R되돌린 메시지 재실행(※ Git 필요)
/compact/summarizeCtrl+X C세션 요약/압축
/connect--AI Provider(모델 제공자) 추가
/models-Ctrl+X M모델 목록
/sessions/resume, /continueCtrl+X L세션 목록/전환
/themes--테마 목록
/editor-Ctrl+X E외부 에디터로 긴 메시지 작성
/export-Ctrl+X X대화 Markdown 내보내기
/share-Ctrl+X S대화 공유 링크 생성(공개 주의)
/unshare--공유 해제
/init-Ctrl+X IAGENTS.md 생성/업데이트
/details-Ctrl+X D툴 실행 상세 표시 토글
/thinking--사고 과정 표시 토글
/exit/quit, /qCtrl+X Q종료

4.2 특수 문법(프롬프트에 직접 사용)

  • @path/to/file : 파일 참조(내용이 모델로 전송됨 → 비밀정보 주의)
  • !command : 셸 명령 실행(출력도 모델로 전송됨 → 비밀정보/파괴적 명령 주의)

4.3 커스텀 명령 만들기(OpenCode)

OpenCode는 마크다운 파일로 커스텀 슬래시 명령을 만들 수 있습니다.

  • 프로젝트 단위:
  • <PROJECT_DIR>/.opencode/commands/
  • 전역 단위(Windows):
  • %APPDATA%\\opencode\\commands\\

형식(YAML frontmatter + 본문 템플릿):

---
description: Brief description
model: optional-model
agent: optional-agent
---

여기에 명령 템플릿을 작성합니다.
!git diff

파일명(확장자 제외)이 명령명이 됩니다.

4.4 설정 파일(OpenCode)

OpenCode는 설정을 두 층으로 둡니다.

  • 전역 설정(Global): ~/.opencode/config.json
  • 프로젝트 설정(Project): 프로젝트 루트의 .opencode 파일

공식 문서: https://www.opencode.asia/configuration/

예시:

// ~/.opencode/config.json
{
  "default_provider": "anthropic",
  "theme": "dark",
  "auto_update": true,
  "telemetry": false
}
# .opencode (프로젝트 루트)
ignore:
  - "dist/"
  - "*.lock"

context_window_limit: 128000

5) Claude Code — 슬래시 명령 “전체 목록”을 다루는 가장 안전한 방법

Claude Code는 공식 문서 표가 존재하지만, 페이지에도 적혀 있듯이 환경에 따라 노출이 달라집니다.

따라서 ‘내 환경에서’ 빠짐없이 확인하려면:

1) 세션에서 / 입력 → 전체 명령 목록 표시 2) / + 몇 글자 입력 → 필터링(검색) 3) 명령 설명을 확인하고 실행

공식 레퍼런스(전체 표): https://code.claude.com/docs/en/commands

이 문서가 제공하는 것은:

  • “명령들이 어떤 범주로 나뉘고, 일반 사용자가 무엇을 언제 쓰면 좋은지”
  • “설정/권한/메모리/훅/스킬/플러그인/MCP가 실제로 어떻게 연결되는지”

6) Claude Code의 핵심 설정 방식(메모리/권한/훅)

6.1 메모리: CLAUDE.md

Claude Code는 보통 다음 층위로 규칙/메모리를 누적합니다.

  • 사용자 전역: ~/.claude/CLAUDE.md
  • 프로젝트: ./CLAUDE.md
  • (선택) 개인용 프로젝트 로컬: ./CLAUDE.local.md (보통 gitignore)

CLAUDE.md에는 다음이 들어가면 좋습니다.

  • 빌드/테스트 명령
  • 기술스택, 폴더 구조
  • 금지사항(any 금지 등)
  • 브랜치 전략(예: main 직접 커밋 금지)

6.2 권한(Permissions)

Claude Code는 “AI가 무엇을 실행할 수 있는지”를 규칙으로 관리합니다. 예를 들어 Bash를 전부 허용하지 않고, git 관련만 허용하는 식입니다.

6.3 훅(Hooks)

훅은 “특정 이벤트(툴 실행 전/후 등)에 자동으로 명령을 실행”하는 기능입니다. 포맷터/린터를 자동으로 돌리거나 위험 명령을 차단하는 데 씁니다.

7) 스킬(Skills)·플러그인(Plugins)·MCP(외부연결)

7.1 스킬(Skills)

  • 스킬은 “자주 하는 작업을 재사용 가능한 워크플로우로 묶은 것”입니다.
  • 호출 방식: /skill ... 형태(환경에 따라), 또는 자동 트리거

7.2 플러그인(Plugins)

  • 플러그인은 여러 구성요소(슬래시 명령 + 스킬 + 훅 + 에이전트 + MCP 등)를 하나로 묶어 배포합니다.

7.3 MCP

  • MCP는 깃헙/DB/노션/슬랙/피그마 같은 외부 시스템을 AI에 연결하는 표준입니다.
  • 보안상 “토큰/자격증명”을 안전하게 다루는 설정이 중요합니다.

10) “진짜 실전” 체크리스트(일반 사용자용)

1) 명령이 기억 안 나면: / 입력해서 목록을 연다(가장 빠르고, 가장 ‘빠짐없음’) 2) 규칙/지시사항을 계속 적용하려면: CLAUDE.md(Claude Code) 또는 AGENTS.md(OpenCode)를 쓴다 3) 반복 작업은: 커스텀 명령(마크다운 파일)로 만든다 4) 외부 서비스 연동은: MCP를 고려한다(토큰/권한 주의) 5) 비밀정보: @ 파일 참조, ! 명령 출력에 섞이지 않게 한다

부록 A) 같은 폴더의 참고 문서

  • claude-code-완전가이드.md — Claude Code 제품 중심의 매우 상세한 설명(설치, 플래그, 설정, 훅, 스킬, MCP, 플러그인 등)
  • opencode-슬래시명령-툴-설정-완전가이드.md — 이 환경(OpenCode Tool 인터페이스)에서의 툴/스킬 인벤토리 중심
