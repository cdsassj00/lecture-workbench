OpenCode/Claude Code 슬래시 명령·툴·설정 완전 가이드 (일반 사용자용)

이 문서는 (1) Claude Code에서 쓰는 슬래시 명령/확장 개념과 (2) OpenCode(툴 호출 기반) 환경에서 실제로 호출 가능한 Tool/Skill/Command를 한 번에 이해할 수 있도록 정리한 “참조용” 가이드입니다. 같은 폴더의 claude-code-완전가이드.md가 Claude Code 제품 중심(설치/CLI/슬래시 명령)이라면, 이 문서는 OpenCode 툴 인터페이스 중심(무엇을 어떤 형식으로 호출할 수 있는지)에 초점을 둡니다.

0) 용어 1분 정리

  • 슬래시 명령(/...): 대화 중 /로 여는 메뉴/명령. (Claude Code UI/CLI에서 주로 등장)
  • Tool(툴): AI가 “실제로 실행”할 수 있는 기능(터미널 실행, 파일 읽기/수정, 검색, LSP 등). 이 문서의 Tool 레퍼런스에 있는 것들.
  • Skill(스킬): 특정 작업을 더 잘하도록 “사용설명서/워크플로우/리소스”를 묶어 둔 확장. 필요 시 로드해 사용.
  • Command(커맨드): 보통 슬래시 명령으로 보이지만, 시스템 내부에선 스킬/플러그인으로 제공되는 경우가 많음.
  • Agent(에이전트)/Subagent(서브에이전트): 역할이 분리된 작업자(탐색, 문서조사, 디버깅 등). task(...)로 실행.
  • 세션(Session): 대화 기록 단위. 이어하기/검색/정보 조회 가능.

1) “다른 코딩툴과 다른 점”을 OpenCode 관점으로 요약

일반적인 코딩 AI(자동완성/채팅)와 달리, OpenCode/Claude Code 계열은 아래가 핵심입니다.

1) 툴 실행 기반: 단순 답변이 아니라 파일을 읽고, 검색하고, 수정하고, 명령을 실행할 수 있음 2) 권한·안전 장치: 어떤 툴을 어떤 범위에서 허용할지 제어 가능(특히 Bash, 파일 쓰기) 3) 확장 생태계: 스킬/플러그인/MCP로 반복 워크플로우를 패키징 가능 4) 에이전트 분업/병렬화: 탐색/리서치/테스트 등 병렬로 돌려 전체 처리 시간을 줄임 5) 영구 메모리/규칙(CLAUDE.md): 프로젝트 규칙/금지사항/작업 방식이 누적됨

2) 시작하기(이 환경에서 “명령을 실행”하는 두 가지 방식)

A. Claude Code(제품)에서의 실행

  • 대화창에서 /로 슬래시 명령 실행
  • @파일경로로 파일 참조, !명령으로 셸 명령 실행

이 부분은 claude-code-완전가이드.md에 상세합니다.

B. OpenCode(툴 호출 기반)에서의 실행

이 환경에서는 “슬래시 명령”이 곧바로 보이지 않을 수 있고, 대신 Tool 호출로 동작합니다.

  • 예: 파일 찾기 → glob(...)
  • 예: 파일 읽기 → read(...)
  • 예: 내용 검색 → grep(...)
  • 예: 수정 → apply_patch(...)
  • 예: 터미널 실행 → bash(...)
  • 예: 에이전트 위임 → task(...) + 완료 후 background_output(...)
  • 예: 스킬/커맨드 로드 → skill(name="...")

3) 설정/메모리(“어디에 규칙을 적어 두면 계속 적용되나?”)

3.1 CLAUDE.md (프로젝트/사용자 규칙)

  • 사용자 전역 규칙: ~/.claude/CLAUDE.md
  • 프로젝트 규칙: ./CLAUDE.md

현재 환경에서 확인된 전역 규칙:

  • 모든 피드백/응답은 한국어로 작성

프로젝트 규칙(예: 브랜치 전략 등)은 프로젝트의 CLAUDE.md에 정의됩니다. (예: main 직접 커밋 금지, claude-dev에서만 작업)

3.2 (프로젝트 예시) 환경변수

프로젝트에 따라 .env/플랫폼 설정(Vercel 등)로 관리됩니다. 예시:

VITE_SUPABASE_URL
VITE_SUPABASE_ANON_KEY
VITE_JUPYTER_KERNEL_URL
VITE_TOSS_CLIENT_KEY
ANTHROPIC_API_KEY

4) Tool 레퍼런스(이 환경에서 호출 가능한 “모든” 툴)

아래는 현재 환경에 노출된 Tool 전체 목록입니다. 사용자는 직접 “툴을 호출”하지 않아도 되지만, 어떤 일이 가능한지(=AI가 무엇을 할 수 있는지)를 이해하는 데 필요합니다.

4.1 사용자에게 질문(선택지 UI)

  • question(...): 진행 중 사용자 선택이 필요할 때(옵션 선택형 질문)

4.2 터미널/명령 실행

  • bash(...): 비대화형 쉘 명령 실행(예: git, npm, pytest 등)
  • interactive_bash(...): tmux 기반의 대화형 TUI 앱(예: vim, htop) 제어

4.3 파일/디렉토리 읽기·수정·검색

  • read(...): 파일 또는 디렉토리 읽기(디렉토리는 목록)
  • glob(...): 파일 패턴 매칭(**/*.ts 등)
  • grep(...): 파일 내용 정규식 검색
  • apply_patch(...): 안전한 패치 형식으로 파일 추가/수정/삭제

4.4 LSP(언어 서버) 기반 코드 인텔리전스

  • lsp_goto_definition(...)
  • lsp_find_references(...)
  • lsp_symbols(...)
  • lsp_diagnostics(...)
  • lsp_prepare_rename(...)
  • lsp_rename(...)

4.5 AST 기반 구조 검색/치환(ast-grep)

  • ast_grep_search(...)
  • ast_grep_replace(...)

4.6 웹/외부 문서·예제 탐색

  • webfetch(...): URL 내용을 가져와(마크다운/텍스트/HTML) 반환
  • websearch_web_search_exa(...): 웹 검색 결과 요약
  • grep_app_searchGitHub(...): GitHub 공개 저장소에서 코드 패턴 검색
  • context7_resolve-library-id(...): Context7 라이브러리 ID 해석
  • context7_query-docs(...): Context7 문서/예제 질의

4.7 에이전트(서브에이전트) 실행

  • task(...): explore/librarian/oracle 등 서브에이전트 실행(동기/비동기)

4.8 백그라운드 태스크 관리

  • background_output(...): 백그라운드 태스크 결과 조회
  • background_cancel(...): 특정 태스크 취소(전체 취소는 권장되지 않음)

4.9 세션(대화 기록) 관리

  • session_list(...): 세션 목록
  • session_read(...): 세션 메시지 읽기
  • session_search(...): 세션 전체 검색
  • session_info(...): 세션 메타데이터

4.10 미디어/간단 추출

  • look_at(...): 이미지/PDF 등에서 “대략적” 정보 추출(정확한 읽기는 read 권장)

4.11 Todo(작업 체크리스트)

  • todowrite(...): 구조화된 작업 목록 생성/업데이트

4.12 스킬/커맨드 로드

  • skill(...): 스킬 또는 “슬래시 커맨드”에 대응하는 항목을 로드/호출

4.13 병렬 툴 실행 래퍼

  • multi_tool_use.parallel(...): 서로 독립인 툴 호출을 병렬로 실행

7) 자주 쓰는 “레시피”(일반 사용자 기준)

레시피 1: 코드에서 특정 문자열/패턴 찾기

1) 파일 후보 찾기: glob("**/.ts") 2) 내용 검색: grep(pattern, include=".ts") 3) 필요한 파일만 read(...)

레시피 2: 안전하게 파일 수정하기

  • 작은 수정은 apply_patch(...)로 “변경점만” 적용
  • 큰 수정은 먼저 read(...)로 컨텍스트 확보 후 패치

레시피 3: 병렬 탐색(시간 절약)

  • 검색/조사 성격 작업은 task(subagent_type="explore"|"librarian", run_in_background=true)로 여러 개 병렬 실행
  • 완료 후 background_output(...)로 결과 회수
