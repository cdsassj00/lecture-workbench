STEP 01 · PROMPT DESIGN

ChatGPT · Gemini · Claude로 Lovable 프롬프트 만들기

Lovable에 바로 입력하기 전에, 먼저 범용 AI 챗봇에 Lovable용 프롬프트 초안을 요청합니다. 기획 의도를 구체화하고 빠뜨린 기능을 잡아내는 데 효과적입니다. 이 단계는 Lovable 크레딧을 전혀 소모하지 않습니다.

ChatGPT

chat.openai.com · GPT-5.5 Instant
구조적 기능 명세 작성에 강함

Gemini

gemini.google.com · Gemini 3 Pro
Google 계정 즉시 사용 · 간결한 출력

Claude

claude.ai · Sonnet 4.6
UX 흐름 설계와 화면 구조 설명에 강함

AI에게 프롬프트를 요청하는 방법

아래 틀에 본인의 아이디어를 채워 ChatGPT·Gemini·Claude 중 하나에 입력합니다.

→ 공통 입력 템플릿 아이디어만 교체
저는 [아이디어 설명]을 만들고 싶습니다. 예를 들어 제가 만들고 싶은 것은: - [핵심 기능 1] - [핵심 기능 2] - [핵심 기능 3] 이걸 Lovable이라는 AI 바이브코딩 툴에서 만들려고 합니다. Lovable에 입력할 프롬프트를 작성해주세요. 조건: - 백엔드는 Lovable 내장 Cloud(Supabase 기반)를 사용합니다 - 로그인/인증은 사용하지 않습니다 (누구나 바로 사용 가능한 앱) - DB에 저장해야 할 데이터 구조도 포함해주세요 - 화면 목록도 명시해주세요 - 한국어로 작성해주세요

아이디어 유형별 예시

예시 A 사진 갤러리
저는 사진 갤러리 웹앱을 만들고 싶습니다. 누구나 사진을 업로드하고 카드 그리드로 갤러리를 보는 앱입니다. 로그인 없이 바로 사용할 수 있어야 합니다. Lovable이라는 AI 바이브코딩 툴에 입력할 프롬프트를 달라고 요청합니다. 백엔드는 Lovable 내장 Cloud를 쓰고, 사진 Storage 업로드 기능이 필요합니다. 한국어로 써주세요.
예시 B 할 일 관리 앱
저는 할 일 관리 앱을 만들고 싶습니다. 할 일 추가, 완료 체크, 삭제 기능이 핵심입니다. 로그인 없이 바로 사용 가능하고, 데이터는 DB에 저장되어야 합니다. Lovable이라는 AI 바이브코딩 툴에 입력할 프롬프트를 써주세요. Lovable Cloud 내장 백엔드를 씁니다. 한국어로 써주세요.
예시 C 게스트북·방명록
저는 간단한 게스트북(방명록) 웹앱을 만들고 싶습니다. 닉네임과 메시지를 입력하면 방명록에 표시되는 앱입니다. 비로그인도 글을 쓸 수 있지만, 로그인한 사람만 삭제 권한이 있습니다. Lovable 바이브코딩 툴에 입력할 프롬프트를 써주세요. Lovable Cloud 내장 백엔드 사용 기준입니다.
✓ AI가 프롬프트를 생성해주면

그대로 Lovable에 붙여넣어도 되고, 마음에 들지 않는 부분은 직접 수정해도 됩니다. AI가 준 프롬프트를 다시 다른 AI에게 검토받는 것도 좋습니다. 예: "이 Lovable 프롬프트에서 빠진 기능이 있으면 추가해줘"

STEP 02 · PROJECT START

Lovable 프로젝트 생성 및 초기 빌드

2-1. 프로젝트 생성

  1. lovable.dev 로그인 → 대시보드 → + New Project 클릭
  2. 프로젝트 이름 입력 — 영문 권장 (GitHub 저장소 이름이 됨)
  3. 빈 에디터 하단 채팅창에 AI로 만든 프롬프트 붙여넣기 → 전송

2-2. Lovable 에디터 핵심 모드

모드용도크레딧
Agent Mode 기본프롬프트 입력 → AI가 코드 자동 생성·수정. 모든 기능 구현에 사용소모 ●
Chat Mode코드 작성 없이 질문·설계·오류 원인 분석. 대화 전용무료 ○
Plan Mode복잡한 요청 전 단계별 계획 먼저 제시. 승인 후 실행계획 무료 / 실행 소모
Visual EditsFigma처럼 UI 요소 클릭 후 색상·여백·폰트 직접 수정무료 ○
Dev ModeLovable 내에서 코드 직접 열람·편집 (2026.05 신규)무료 ○
⚠ 크레딧 절약 원칙

오류가 반복되면 "Try to Fix" 버튼을 계속 누르지 마세요. 크레딧만 소모됩니다. 오류 발생 시 → Chat Mode에서 원인 파악 → Plan Mode로 수정 계획 수립 → 실행. 아니면 히스토리에서 롤백으로 정상 버전으로 되돌리세요.

2-3. 추가 기능 요청 프롬프트 작성법

초기 생성 이후 기능을 하나씩 추가할 때는 짧고 명확하게 요청합니다.

Good 구체적·단일 요청
게시글 목록 화면에서 각 카드 클릭 시 상세 보기 페이지(/posts/:id)로 이동하게 해줘.
Bad 한 번에 너무 많은 요청
// 이렇게 하면 AI가 중간에 실수할 확률 높음 게시글 CRUD 만들고, 댓글 기능도 넣고, 좋아요 버튼도 추가하고, 파일 업로드도 해줘.

기능은 한 번에 하나씩 요청하는 것이 크레딧 효율과 안정성 모두에 유리합니다.

STEP 03 · LOVABLE CLOUD BACKEND

Lovable Cloud 내장 백엔드 설정

Lovable Cloud는 Supabase 기술 기반의 내장 백엔드입니다. 별도 Supabase 계정 없이 Lovable 에디터 안에서 DB·Storage를 바로 사용할 수 있습니다. 이번 과제에서는 인증 없이 개발합니다.

✓ Lovable Cloud의 장점

외부 계정 연결 없이 즉시 사용 가능. 에디터 내 패널에서 테이블 확인. Storage도 프롬프트 한 줄로 설정. 코드 내 환경변수 자동 처리.

3-1. Cloud 활성화

  1. Lovable 에디터 상단 + 버튼 → Cloud 선택
  2. Enable Lovable Cloud 클릭 → 내장 DB·Storage 자동 생성
  3. 에디터 사이드패널에 Database / Storage 탭이 생기면 완료
⚠ 2026년 3월 이후 변경사항

2026년 3월 24일부터 신규 Cloud 프로젝트에서 Test/Live 환경 분리 기능은 제공되지 않습니다. 기존 프로젝트는 계속 사용 가능하지만, 지금 새로 만드는 프로젝트는 단일 환경으로 운영됩니다.

3-2. 기능별 활성화 (프롬프트)

Cloud 활성화 후 필요한 기능을 Agent Mode에서 프롬프트로 요청합니다.

데이터베이스 (DB 테이블 생성)

Agent Mode 예시: 할 일 앱
Lovable Cloud DB에 tasks 테이블을 만들어줘. 컬럼: id(UUID, 기본키), title(텍스트), completed(불리언, 기본값 false), created_at(타임스탬프) 누구나 데이터를 읽고 쓸 수 있어야 해. 로그인 없이 사용하는 앱이야.

생성 후 에디터 사이드패널 → Database 탭에서 테이블 구조 확인 가능합니다.

Storage (파일·이미지 업로드)

Agent Mode 예시: 사진 갤러리
Lovable Cloud Storage에 'photos' 버킷을 만들고, 사진 업로드 버튼을 메인 화면에 추가해줘. 업로드 후 갤러리 그리드에 바로 표시해줘. 로그인 없이 누구나 업로드 가능하게 해줘.

Edge Function (서버리스 로직)

Agent Mode 예시: 자동 처리 함수
게시글이 저장될 때 글자 수를 자동으로 계산해서 DB의 char_count 컬럼에 저장하는 Edge Function을 만들어줘. Lovable Cloud Edge Function을 사용해.
외부 API Key가 필요한 경우 (예: OpenAI)

Lovable 에디터 상단 + → Cloud → Secrets → + 버튼으로 키 추가. 코드에 직접 키를 입력하면 GitHub에 노출되므로 반드시 Secrets를 사용하세요. Secrets에 저장한 이름(OPENAI_KEY)과 함수 코드 내 참조 이름이 대소문자까지 동일해야 합니다.

STEP 04 · GITHUB INTEGRATION

GitHub Integration 연동

Lovable ↔ GitHub는 양방향 자동 동기화입니다. Lovable에서 변경하면 GitHub 커밋이 자동 생성됩니다. 반대로 GitHub에서 코드를 수정해도 Lovable에 반영됩니다.

4-1. 연동 절차

  1. 에디터 우측 상단 GitHub 아이콘 클릭 (또는 상단 메뉴 → GitHub)
  2. Connect to GitHub → 브라우저 팝업에서 GitHub 로그인·권한 허용
  3. 저장소 이름 확인 (프로젝트 이름 자동 입력) → Create Repository
  4. "Connected to GitHub" 표시 확인
  5. github.com → 본인 계정 → 저장소 검색 → 코드 파일 목록 확인

4-2. 제출용 GitHub URL 확인

URL 형식 확인

github.com 접속 → 해당 저장소 클릭 → 브라우저 주소창의 URL 복사

형식: https://github.com/본인아이디/저장소이름

저장소가 Public으로 설정되어 있는지 확인 (기본값이 Public이지만 확인 필요)

⚠ 팝업 차단 주의

GitHub 연동 시 팝업이 열려야 합니다. 브라우저에서 팝업이 차단되면 주소창 오른쪽 팝업 차단 아이콘 클릭 → 허용 후 재시도합니다.

4-3. Dev Mode로 코드 직접 확인

에디터 상단 Dev Mode 토글 → 생성된 TypeScript/React 코드를 에디터 내에서 직접 볼 수 있습니다. 구경하거나 간단한 텍스트 수정에 사용하세요. 크레딧 소모 없음.

STEP 05 · PUBLISH

Publish — 배포 URL 확보

5-1. 배포 전 점검

  • 미리보기 패널에서 핵심 기능 사용 → 데이터 반영 흐름 테스트
  • 빨간 에러 콘솔이 있으면 Chat Mode에서 원인 파악 후 수정
  • DB 데이터가 실제 저장되는지 에디터 → Database 탭에서 확인

5-2. Publish 실행

  1. 에디터 우측 상단 Publish 버튼 클릭
  2. Cloud가 연결된 경우 → Security Scan 자동 실행 (2026.05 신기능). 경고가 있으면 내용 확인. 심각하지 않으면 무시 가능
  3. Publish 클릭 → 빌드 1~3분 대기
  4. 완료 후 배포 URL 표시: https://프로젝트이름.lovable.app
  5. 배포 URL 복사 → 이것이 제출용 Lovable URL

5-3. 배포 후 실제 테스트

  • 시크릿 창(비공개 탭)에서 배포 URL 접속 — 캐시 없는 상태로 테스트
  • 핵심 기능 사용 → 데이터가 실제로 저장되는지 확인
  • 모바일 브라우저에서도 레이아웃 확인

5-4. 재배포

기능을 수정하거나 추가한 뒤에는 Publish 버튼을 다시 클릭하면 됩니다. 같은 URL 그대로 업데이트됩니다.

TROUBLESHOOTING

자주 발생하는 문제 해결

데이터가 저장되지 않는다

원인 및 해결

원인 A · 테이블의 행 수준 보안(RLS) 정책이 public 접근을 막고 있음 → Chat Mode: "tasks 테이블에 데이터가 저장 안 되는 이유를 분석해줘. 로그인 없이 누구나 쓸 수 있어야 해"

원인 B · Cloud가 활성화되지 않은 상태 → 에디터 + → Cloud → 활성화 여부 확인

원인 C · 테이블 컬럼명이 코드와 불일치 → Database 탭에서 실제 컬럼명 확인 후 프롬프트로 수정 요청

AI가 같은 오류를 반복한다 (루핑)

해결 순서

1. 히스토리 패널에서 마지막 정상 버전으로 롤백

2. Chat Mode: "지금 [오류 내용]이 발생하고 있어. 원인이 뭔지 분석해줘"

3. Plan Mode: "이 오류를 고치는 단계별 계획을 세워줘. 바로 코드 수정하지 말고 계획만 먼저 보여줘"

4. 계획 확인 후 승인 → 실행

GitHub 연동이 안 된다

해결 순서

1. 브라우저 팝업 허용 설정 확인 (주소창 우측 팝업 차단 아이콘)

2. 새 탭에서 github.com 접속 → 로그인 상태 확인

3. Lovable 에디터 새로고침 후 재시도

4. GitHub → Settings → Applications → Lovable 권한 재승인

Edge Function이 오류를 반환한다

해결 순서

에디터 → Cloud 패널 → Functions → 해당 함수 → Logs 탭에서 오류 내용 확인

Secrets 이름이 코드와 대소문자까지 동일한지 확인 (OPENAI_KEY ≠ openai_key)

Chat Mode: "Edge Function [함수이름]이 오류를 반환하고 있어. Logs에 [오류내용]이라고 나오는데 원인이 뭔지 알려줘"

  • DB / Storage / Edge Function 중 1개 이상이 실제로 작동한다 (인증 불필요)
  • GitHub Integration을 연결했고 저장소에 코드가 push되어 있다
  • GitHub 저장소 URL을 복사해 두었다 (github.com/아이디/저장소명)
  • Publish를 실행하여 .lovable.app 배포 URL을 확보했다
  • 시크릿 창에서 배포 URL 접속 → 핵심 기능 정상 동작 확인
  • 구글시트에 소속·이름·Lovable URL·GitHub URL 4개 항목 입력 완료
✓ 완성도보다 기획 의도가 핵심

크레딧 한계로 미완성이 되어도 감점 없습니다. Publish URL과 GitHub URL 2개가 있고, 만들고자 했던 것이 무엇인지 보이면 충분합니다.
