Assignment 01 · Day 1

Hugging Face SLM 모델 조사 보고서

GitHub 계정을 생성하고, Hugging Face에서 1B~7B 규모의 Small Language Model 1개를 직접 분석해 세련된 HTML 기술 리포트로 정리합니다.

01과제 개요

공개된 소형 언어모델을 직접 탐색하고, 모델 카드와 설정 파일(config.json · tokenizer.json · model.safetensors 등)을 읽어 핵심 내용을 정리하는 실무형 과제입니다.

개발자가 아니더라도 AI 전문인재라면 어떤 모델이 어떤 데이터로 학습되었고, 어떤 방식으로 튜닝되었으며, 어떤 업무·산업 영역에 적합한지 판단할 수 있어야 합니다.

핵심 방향 · 단순 검색 결과 붙여넣기가 아니라, 선택한 SLM을 하나의 기술 제품처럼 분석하고 다른 직원이나 의사결정자가 이해할 수 있는 수준의 깔끔한 HTML 리포트로 정리합니다.

02모델 선택 기준

  • 플랫폼 — Hugging Face
  • 모델 규모 — 1B 이상 7B 이하의 Small Language Model
  • 권장 모델 — 파인튜닝 모델, 특정 산업 도메인 모델, 공공업무 활용 가능 모델
  • 모델 유형 — Base / Instruct / Chat / Domain-specific 등
  • 제외 대상 — 이미지 전용, 음성 전용, 임베딩 전용, 7B 초과 모델

03조사해야 할 내용

  • 모델 기본 정보 · 모델명, Hugging Face 주소, 개발 기관, 라이선스
  • 학습 데이터 · 사전학습 데이터, 공개 여부, 산업·언어·도메인 데이터 포함 여부
  • 튜닝 방식 · Instruction Tuning · SFT · RLHF · DPO · LoRA · QLoRA 등 적용 여부
  • 파라미터 구조 · 파라미터 수, 아키텍처, 컨텍스트 길이, 레이어·헤드·hidden size
  • 모델 카드 · 용도, 한계, 평가 결과, 사용 예시
  • 파일 구성 · config.json, tokenizer, safetensors, generation_config.json 등의 의미
  • 활용 가능성 · 행정·민원·문서 분석·질의응답·요약·RAG 등에 어떻게 활용 가능한지

04제출 절차

  • ① GitHub 계정 생성 — github.com에서 Sign up
  • ② Repository 생성 — 이름은 slm-model-report-번호-이름 형식 권장 · Public 설정
  • ③ 우측 「보고서 템플릿 다운로드」 버튼으로 HTML 양식 받기
  • ④ 브라우저에서 템플릿 열기 → EDIT 버튼 클릭 → 9개 섹션 작성
  • ⑤ Ctrl+S 또는 「HTML 저장」 버튼으로 번호_이름_slm_report.html 자동 다운로드
  • ⑥ 저장된 HTML을 GitHub Repository에 업로드
  • ⑦ 최종 제출 시 GitHub Repository 주소 운영진에 공유
