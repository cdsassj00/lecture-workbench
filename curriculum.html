기획하는 거점리더를 위한
8주 AI 실전 여정

기획자가 그린 그림이 있어야 에이전트의 구현이 시작됨.

본 과정은 기술자를 양성하는 과정이 아님. 공공 현장의 거점리더, 즉 자기 부서의 문제를 가장 잘 이해하고 동료에게 AI 활용을 코칭·확산할 수 있는 인력을 양성함을 목적으로 함. 수강생은 REST API의 내부 구현이나 벡터 DB의 알고리즘 세부를 직접 다루지는 않음. 다만 산출물의 적정성을 검증하고 동료에게 설명할 수 있을 정도의 최소 원리는 보유함. 핵심 역량은 "어떤 재료를 어떤 순서로 엮으면 이 문제가 풀릴까"를 아키텍처로 그릴 수 있는 능력임. 구현은 에이전트가 담당함.

전체 8회차는 세 개의 구간으로 구성됨. 1~3회차는 AI 도구·서비스 생태계를 지도화해 워크플로우를 기획하는 구간, 4~5회차는 외부망 AI 어시스트로 코드를 받아 내부망에서 실행하는 공공 현실 기반의 바이브 코딩 구간, 6~8회차는 에이전트 도구로 본격 개발에 들어가 RAG·웹서비스·API·MCP를 조합해 풀스택 서비스를 기획·구축하는 구간임.

Design Philosophy

· 심층 구현은 에이전트가 담당함. 거점리더는 검증·판단·설명에 필요한 최소 원리를 보유함.

· 9가지 공공 구현 유형을 어시스트 버전과 에이전트 버전으로 두 차례 경험함.

· 모든 회차는 문제 제시 → 도구 조합 → 산출물 → 공유·축적 사이클로 종결됨.

· 수강생의 최종 목표는 비즈니스 모델 캔버스처럼 AI 아키텍처를 1장에 기획하는 역량임.

Session 01AI 에이전트 시대의 개막과 공공 구현 지형도공공 AI 거점리더의 기술 지형과 환경 세팅

과정의 출발점에서 수강생은 두 가지를 얻어야 함. 첫째는 현재 서 있는 기술적 위치에 대한 감각임. Bard·PaLM에서 시작해 GPT 시대를 거쳐 에이전트 시대로 넘어온 진화사를 한 호흡에 정리하고, MicroGPT를 뜯어보며 트랜스포머의 Q·K·V·멀티헤드 구조를 시각적으로 체감함. 프롬프트가 컨텍스트로, 컨텍스트가 하네스(프로젝트 규칙·스킬·도구·승인이 한 덩어리로 작동하는 에이전트 실행 환경)로 진화한 맥락도 같은 흐름에서 짚음. 심층 학습은 지양하며, 과정의 필요성을 납득시키는 것이 목적임.

둘째는 과정 종료 시점에 만들 수 있는 것에 대한 구체적 지도임. 공공 현장에 도입 가능한 '민첩한 AI' 9가지 구현 유형을 개괄 수준으로 제시하며, 각 유형이 어느 회차에서 어떻게 다뤄지는지 미리 인지하도록 구성함.

Session 02프론티어 모델 심층 탐구와 AI 도구 생태계 50선AI 도구·서비스 생태계를 지도화하는 시간

거점리더의 무기는 보유 도구의 개수가 아니라 새 서비스 출현 시 당황하지 않는 적응력임. 본 회차는 해당 적응력을 체화하기 위해 두 층위로 구성됨. 앞 절반은 플래그십 모델의 숨은 킥을, 뒤 절반은 10 카테고리 × 대표 5개 = 50선의 지도화 투어임.

앞 절반에서는 ChatGPT·Claude·Gemini·Perplexity를 '남들이 잘 모르는 킥' 중심으로 파헤침. Grok·Z.AI 대안도 함께 다룸. DeepSeek·Qwen·MiniMax 등 중국 계열은 '이런 것이 존재함' 수준으로 사이트 접속만 체험하되 공공 데이터 업로드는 엄격 금지함. 이어서 Ollama·LM Studio·AnythingLLM·MSTY를 직접 설치해 한 번씩 구동해봄. 해당 경험은 5회차 VBA 엑셀 AI 함수와 7회차 Ollama RAG로 연결되는 복선임.

뒤 절반은 50개 사이트를 10개 카테고리로 투어함. 빠른 호흡으로 쏟아붓되 각 도구의 UI 공통 패턴을 관찰하게 함. '새 서비스 출현 시 빠른 파악 루틴'을 체크리스트로 도출하는 것이 본 회차의 실질 목표임.

Session 03메타 프롬프트 설계와 AI 서비스 워크플로우 오케스트레이션프롬프트 기법을 외우지 않고 워크플로우를 조립함

본 회차 전반부는 메타 프롬프트임. 역할·맥락·제약·예시·검증 같은 전통 기법을 암기하는 방식이 아님. 생성 AI에게 '내 상황·목표·제약을 전달하니 최적 프롬프트를 설계해달라'고 요청해 프롬프트 자체를 AI에게 생성시키는 접근임. 생성된 프롬프트는 다른 모델에 교차 투입해 검증·개선함.

핵심은 프롬프트 공유 문화임. 수강생 20~40명이 각자 자주 쓰는 프롬프트 2개씩만 메타 프롬프트로 생성해 공용 저장소에 등록해도 80개 시드 라이브러리가 확보됨. 동료 프롬프트를 자기 부서 맥락에 맞게 변형·재사용하는 훈련, 실패 프롬프트도 실패 사유와 함께 공유하는 훈련이 거점리더 역할의 본질과 직결됨.

회차의 메인은 AI 서비스 워크플로우 오케스트레이션임. 본 단계에서는 에이전트 툴을 사용하지 않음. 2회차에서 지도화한 AI 서비스들을 여러 개 연결해 하나의 과제를 해결하는 경험이 학습 목표임. 강사가 6가지 워크플로우 타입을 전부 시연한 후, 수강생은 그중 하나를 재현하거나 자기 현업 과제를 직접 조립함.

  • 시스템 프롬프트 · 프로젝트 지침 · Custom GPT 지침 메타 생성
  • 프롬프트 공유 활동 — 1인 2개 × 20~40명 = 80개 시드 라이브러리
  • 거점리더의 부서 이식 가이드

Session 07RAG 파이프라인과 웹서비스 구축재료로서의 RAG 감각 체득

RAG를 기술자 버전이 아니라 기획자 버전으로 이해하는 회차임. RAG는 '내 문서 창고를 AI가 탐색해 응답하게 하는 구조'이며, 파이프라인은 4단계로 기억함: 넣기·쪼개기·찾기·답하기. 문서를 넣고(파싱), 의미 단위로 자르고(청킹), 질문과 유사한 조각을 꺼내오고(벡터 검색), 해당 조각을 AI에게 전달해 답을 생성함. 리랭크·하이브리드·GraphRAG 같은 고급 기법은 '정확도 향상 옵션' 수준으로 명칭만 인지함.

실습은 에이전트에게 위임함. 'Ollama + 내 PDF 10개로 부서 Q&A 구성해달라'고 요청하고, 완성된 결과물에서 변경 지점별 출력 변화를 실험하는 방식임. 본 경험이 법령 RAG, 민원 매뉴얼 RAG, 회의록 RAG 등 공공 사례 기획으로 확장됨.

웹서비스도 재료로 이해함. 프론트(표시)·백엔드(처리)·저장소 3덩어리만 인지하면 충분함. FastAPI·Express 등 프레임워크 선택은 에이전트가 담당함. 수강생의 역할은 프론트+백엔드+RAG 결합을 아키텍처 그림으로 설계하고 에이전트에 구현을 위임하는 것임.

RAG 파이프라인 — 4단계 (Part A)

  • 넣기 — DeepSeek-OCR, 한컴 PDF 파서, LangChain 로더

  • 찾기 — 벡터 DB (Chroma, FAISS)로 유사 조각 검색
  • 답하기 — 검색된 조각을 AI에 전달해 답 생성
  • 고급 옵션 — 리랭크, 하이브리드, GraphRAG (명칭만)

Session 08API · MCP 기반 멀티 에이전트 아키텍처 기획 및 구축조합의 끝, 기획자로 완성되는 회차

최종 회차는 아키텍처 기획자로 완성되는 자리임. API와 MCP를 비개발자 친화 설명으로 이해하고, 그 위에서 비즈니스 모델 캔버스와 같은 방식으로 AI 아키텍처를 1장에 설계하는 워크숍이 본 회차의 하이라이트임.

API는 '타인이 구축한 기능을 호출해 사용하는 창구'임. 구성 원리는 단순함 — 질의 송신 시 응답 수신. 수강생이 인지해야 할 상식은 세 가지 — 키는 비밀번호이므로 노출 금지, 유료인 경우가 많으므로 쿼터·요금 주의, 간헐적 실패이므로 재시도 필요. REST·헤더·JSON 같은 기술 세부의 코드 작성은 에이전트가 처리함. 거점리더는 어떤 호출이 일어나고 어떤 응답을 받는지 한 줄로 설명할 수 있어야 함.

MCP는 '에이전트가 외부 도구를 연결하는 표준 콘센트'로 이해함. 기존 MCP 서버 카탈로그를 재료 목록처럼 순회하고, 에이전트에 MCP를 연결하는 작업은 설정 1회 수준으로 시연함. MCP 서버 직접 제작도 에이전트에게 위임함. 다만 어떤 도구를 왜 연결했는지 거점리더가 설계 의도를 설명할 수 있어야 함.

이 모든 기반 위에서 AI 아키텍처 캔버스 워크숍이 진행됨. 트리거·재료·처리·시스템 프롬프트·산출물·배포 6칸에 각자의 거점 미션을 1장으로 설계함. 최종 발표는 1회차에 선언한 거점 미션이 어떻게 현실화되었는지 연결하는 자리임.

API 비개발자 친화 설명 (Part A)

  • API = 타인이 구축한 기능을 호출해 사용하는 창구
  • 상식 3종 — 키 보안, 쿼터·요금, 재시도
  • REST·헤더·JSON은 에이전트가 전담
  • 수강생의 실질 질문 — '이 서비스에 API가 존재하는가?'

공공 API 카탈로그 투어 (Part B)

  • OpenWeather · SerpAPI · 국가법령정보 · 기상청 API 허브 · 공공데이터포털 · 행정안전부 표준 API
  • 각 API를 재료 카드로 정리 — 제공 데이터 / 활용 영역
  • 수강생별 '쓸만한 API 1건 발굴' 공유

MCP 비개발자 친화 설명 (Part C)

  • MCP = 에이전트가 외부 도구를 연결하는 표준 콘센트
  • 기존 MCP 서버 카탈로그 투어

1회차에 선언한 거점 미션을 AI 아키텍처 캔버스 6칸(트리거·재료·처리·시스템 프롬프트·산출물·배포)에 1장으로 설계함. 해당 캔버스를 에이전트에 전달해 실제 구현물을 도출하고 검수 노트를 작성함. 거점리더 최종 발표(5~7분) — 1회차 미션과 8회차 산출물의 연결, 현업 복귀 후 동료 2~3명 코칭·전파 계획을 선언함. 코칭·전파 계획에는 대상(코칭 동료 2~3명 직무·관심 유형) · 일정(복귀 후 4주 안 시범 코칭, 8주 안 부서 단위 시연) · 자료(공용 저장소에 업로드한 자기 산출물 + 빠른 파악 루틴 체크리스트) · 후속 점검(3개월 시점 코칭 결과 회고 1장)을 포함함.
