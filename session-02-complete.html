"이 도구가 좋은가"는 의미가 약한 질문이다. "이 흐름의 어느 단계에 이 도구를 끼울 것인가"가 실제로 답할 수 있는 질문이다. 결합 지점을 본다.

FEATURE 01 · PROJECTS
채팅 · 파일 · 지침을
하나로 묶는 작업 공간
프로젝트 안에서는 대화 간 맥락이 자동 유지된다. 파일을 올려두면 이후 모든 대화가 그것을 참조한다. 부서 지침 · 샘플 보고서 · 금칙어 세트를 한 프로젝트에 두면 동료가 진입 비용 없이 같은 기준의 답변을 받는다.
질문 템플릿 5개를 고정 메시지로 상단에 배치하고, 주 1회 업데이트 담당자를 정해두면 자산이 된다.
FEATURE 02 · CUSTOM GPTs
시스템 프롬프트 + 파일 + Actions
= 재사용 단위 "나만의 GPT"
GPT Store에서 남이 만든 것도 가져다 쓸 수 있다. 공공에서는 "민원 분류 GPT" · "예산 검토 GPT" 같은 업무 특화 GPT를 거점리더가 만들어 부서에 배포한다. Actions는 외부 API 호출 기능이지만 8차시 MCP로 대체되는 방향이다.
Custom GPT는 "프롬프트의 패키징"이다. 좋은 프롬프트를 그대로 동료에게 배포하는 가장 빠른 방법.

AXIS 02 · MEMORY
대화에서 사용자 정보를

민감 업무엔 메모리 OFF. 주기적으로 저장된 메모리를 점검한다.

"생각하는 시간"으로

단순 요약·분류에는 일반 모델로 충분하다. 모든 업무에 o3를 쓰면 비용과 속도 모두 무너진다.

데이터가 어디 저장되는가
미국 · 유럽 · 한국 · 로컬 중 어디인가. 공공에서는 이 칸이 가장 먼저 검토된다. 답이 나오지 않으면 다음 칸으로 넘어가지 않는다.

API가 있는가 · MCP 서버는?
다른 도구와 결합 가능성을 판단한다. API·MCP가 없으면 "단품용"으로 분류한다. 있으면 워크플로우 조립 후보.

외부 도구 결합 · 내부망 처리 · 2단계 이식의 3가지 패턴. 세 예시가 보여주는 것은 같은 업무도 보안 수준에 따라 도구가 달라진다는 점. 보안은 피해야 할 제약이 아니라 설계의 첫 축이다.

CSV → 차트
자동 생성
현재 업무: 엑셀 피벗 → 차트 수작업 — 시간 많이 걸림.
결합: CSV 업로드 → Julius에서 차트 즉시 생성.
앞뒤: 데이터 추출 → [Julius] → 보고서 삽입.
보안: Julius 서버 해외 — 비식별화 필수.
개인정보가 포함된 원본은 열·행을 가린 비식별본으로 변환 후 업로드. 10분의 비식별 작업을 포함해도 가치가 있다.
EXAMPLE 02 · WHISPER 로컬 + 회의록
녹음 파일 →
완전 로컬 STT

결합: 녹음 파일 → Whisper 로컬 실행 → 텍스트.
보안: 완전 로컬 · 외부 통신 0.

외부 테스트 →
