ChatGPT 숨은 킥 첫 두 가지 — Projects와 Custom GPTs
ChatGPT에서 거점리더가 먼저 장악해야 할 두 가지 숨은 킥을 다룬다. 첫째, Projects(ChatGPT에서 대화·파일·지침을 하나의 프로젝트로 묶어 두는 작업 공간)는 올려 둔 파일을 이후 모든 대화가 자동 참조한다. 둘째, Custom GPTs(시스템 프롬프트·파일·Actions를 패키징해 재사용하는 "나만의 GPT")는 부서 업무에 특화된 챗봇을 만들어 동료에게 배포하는 수단이다. 거점리더는 둘 다 개인용이 아니라 부서 배포 단위로 세팅한다.
FEATURE 01 · PROJECTS
채팅 · 파일 · 지침을
하나로 묶는 작업 공간
프로젝트 안에서는 대화 간 맥락이 자동 유지된다. 파일을 올려두면 이후 모든 대화가 그것을 참조한다. 부서 지침 · 샘플 보고서 · 금칙어 세트를 한 프로젝트에 두면 동료가 진입 비용 없이 같은 기준의 답변을 받는다.
질문 템플릿 5개를 고정 메시지로 상단에 배치하고, 주 1회 업데이트 담당자를 정해두면 자산이 된다.
FEATURE 02 · CUSTOM GPTs
시스템 프롬프트 + 파일 + Actions
= 재사용 단위 "나만의 GPT"
GPT Store에서 남이 만든 것도 가져다 쓸 수 있다. 공공에서는 "민원 분류 GPT" · "예산 검토 GPT" 같은 업무 특화 GPT를 거점리더가 만들어 부서에 배포한다. Actions는 외부 API 호출 기능이지만 8차시 MCP로 대체되는 방향이다.
Custom GPT는 "프롬프트의 패키징"이다. 좋은 프롬프트를 그대로 동료에게 배포하는 가장 빠른 방법.

빠른 파악 루틴 6단계

17:10~17:30 워크숍에서 도출하는 산출물입니다. 6단계(IN/OUT · FREE · WHERE · API/MCP · INSERT · LINK)를 자기 업무 맥락에 맞춰 5~7항목으로 추리고, 기관 GitHub Org 저장소의 routine-checklist.md로 업로드합니다. 이후 새 AI 서비스가 출현할 때 같은 6단계로 한 칸씩 답을 적어 약 5분 안에 도입 여부를 판단합니다.

STEP 01 · IN → OUT
무엇을 받아 무엇을 내보내는가
입력 형식 · 출력 포맷 · 품질 기준을 한 줄로 적는다. 여기서 막히면 그 도구가 무엇을 하는 도구인지 아직 파악이 안 된 것.
STEP 02 · FREE
무료로 쓸 수 있는가 · 제한은?
일일 크레딧 · 워터마크 · 내보내기 제한 확인. "무료" 표시만 보고 도입 금지 — 실제 제약을 확인하지 않으면 재작업.
STEP 03 · WHERE
데이터가 어디 저장되는가
미국 · 유럽 · 한국 · 로컬 중 어디인가. 공공에서는 이 칸이 가장 먼저 검토된다. 답이 나오지 않으면 다음 칸으로 넘어가지 않는다.
STEP 04 · API · MCP
API가 있는가 · MCP 서버는?
다른 도구와 결합 가능성을 판단한다. API·MCP가 없으면 "단품용"으로 분류한다. 있으면 워크플로우 조립 후보.
STEP 05 · WHERE · USE
내 업무의 어느 단계에 끼우나
현재 업무 흐름의 어느 단계를 대체하거나 강화할 것인가. 업무 흐름을 먼저 단계로 나눈 뒤에 빈칸을 찾는다.
STEP 06 · LINK
이미 쓰는 도구와 어떻게 연결되나
상위 단계 · 하위 단계 도구와 포맷 호환 여부. 파일 형식 · 인증 · 복붙 편의성. 결합이 매끄러우면 실제 사용률이 높다.
