파일시스템 자동화의 3줄 패턴
파일 찾기와 정리는 평균 주 90분의 수작업으로 돌아오는 대표적인 반복 업무입니다. 자동화의 구성은 세 줄입니다. 첫째, pathlib.Path.glob()으로 확장자·날짜 패턴을 수집합니다. 둘째, 정규식이나 조건식으로 대상만 거릅니다. 셋째, shutil.move/copy로 규칙별 폴더에 옮깁니다. 같은 규칙을 3회만 반복해도 작성 시간이 회수되므로, 부서 폴더 한 개의 규칙부터 시작하는 것이 안전한 진입 경로입니다.
CODE민원 PDF 월별 자동 분류
실습예시 — 파일명의 날짜로 월별 폴더 분류
# 민원 PDF를 YYYY-MM 폴더로 자동 이동 from pathlib import Path import shutil, re src = Path("./민원접수") for f in src.glob("*.pdf"): m = re.search(r"(\d{4})-(\d{2})", f.name) if m: dst = src / f"{m[1]}-{m[2]}" dst.mkdir(exist_ok=True) shutil.move(f, dst / f.name) print(f"{f.name}→{dst.name}/")

핵심은 glob(패턴)·Path 연산자·shutil.move 세 줄. 이 세 줄이 공공 반복 업무 상당수를 해결한다.

공공 업무의 "보고 → 회신" 루프는 PDF와 메일 두 조각으로 연결됩니다. 첫째, PDF 처리는 PyMuPDF(fitz) 하나면 대부분 끝납니다 — page.get_text()로 본문, page.find_tables()로 표, doc.insert_pdf로 병합이 모두 해결됩니다. 둘째, 메일은 파이썬 표준 라이브러리 smtplib + email.mime 조합이 표준이며 첨부·HTML 본문·수신자별 맞춤 발송까지 커버합니다. 셋째, 기관 SMTP는 SSL(465)과 STARTTLS(587) 중 정책에 맞는 쪽을 고르고, 스캔 PDF는 별도 OCR 단계를 붙입니다.
PDFPyMuPDF(fitz)
# 민원 PDF에서 3개 필드 추출 → 엑셀 import fitz, re, openpyxl from pathlib import Path wb = openpyxl.Workbook(); ws = wb.active ws.append(["파일", "민원일시", "제목", "요지"]) for p in Path("./pdfs").glob("*.pdf"): doc = fitz.open(p) txt = "".join(pg.get_text() for pg in doc) date = re.search(r"\d{4}-\d{2}-\d{2}", txt) title = re.search(r"제목:\s*(.+)", txt) ws.append([p.name, date.group() if date else "", title.group(1) if title else "", txt[:60]]) wb.save("민원취합.xlsx")

HWP 양식 자동 채우기 — 누름틀 기반
HWP 양식 자동 채우기는 공공 문서 포맷 변환의 기본 블록입니다. 흐름은 네 단계입니다. 첫째, 기존 HWP 양식 파일을 pyhwpx(파이썬에서 한컴 한글 파일(HWP)을 제어하는 라이브러리)의 open()으로 엽니다. 둘째, 이름·부서·일자·금액처럼 변하는 값을 dict로 준비합니다. 셋째, put_field_text(key, value) 한 줄로 누름틀(필드)에 값을 주입합니다. 넷째, "{YYYYMMDD}_{부서}_{이름}.hwp" 같은 파일명 규칙으로 저장합니다. 출장비 정산·교육 수료증·복무 통지 같은 반복 문서가 이 한 패턴으로 수렴합니다.
CODE출장비 정산 양식 자동 생성
# 출장비 정산 HWP 양식 1장 자동 생성 from pyhwpx import Hwp hwp = Hwp(visible=False) hwp.open("./tpl/출장비정산_양식.hwp") data ={ "이름": "김주무관", "부서": "디지털정책팀", "일자": "2026-04-20", "출장지": "세종청사", "금액": "128,500", }for k, v in data.items(): hwp.put_field_text(k, v) fname = f"{data['일자']}_{data['부서']}_{data['이름']}.hwp" hwp.save_as(f"./out/{fname}") hwp.quit()

핵심 한 줄은 put_field_text(key, value). CSV 헤더명과 HWP 누름틀 이름을 맞춰두면 dict 하나가 양식 하나가 된다.

양식을 채운 다음의 마무리 단계는 세 가지 변환으로 정리됩니다. 첫째, 결재·공유용 최종본은 save_as(".pdf")로 일괄 PDF 변환해 편집용 HWP와 배포용 PDF를 분리 보관합니다. 둘째, 회의자료·보고서 조각들은 insert_file 루프로 한 파일에 병합하고 목차·페이지번호는 병합 뒤 한 번만 재생성합니다. 셋째, HWP 안의 표는 get_into_table()로 추출해 openpyxl로 엑셀 파일에 고정 저장함으로써 수치 원본을 아카이브합니다. 이 세 변환이 HWP 실무 체감의 약 90%를 차지합니다.

다섯 개 AI 함수가 공유하는 공통 헬퍼 OllamaCall 한 개를 먼저 작성하고, 이후 다섯 UDF는 프롬프트 한 줄만 다른 wrapper로 추가합니다. 구조는 네 단계입니다. 첫째, VBA(엑셀·한글 등 MS Office와 한컴 오피스에 내장된 매크로 언어)의 MSXML2.XMLHTTP 객체로 http://localhost:11434/api/generate에 POST 요청을 엽니다. 둘째, 사용자 프롬프트를 EscapeJSON으로 감싸{"model":...,"prompt":...,"stream":false}형태로 본문을 만듭니다. 셋째, Ollama(로컬 PC에 LLM을 띄워 주는 무료 도구)의 응답 JSON에서 response 필드만 파싱해 문자열로 반환합니다. 넷째, 이 OllamaCall 함수 하나만 잘 만들면 이후 다섯 UDF는 프롬프트 한 줄만 바꿔 찍어내면 됩니다 — 유지보수가 래퍼 한 지점으로 수렴합니다.
VBA공통 HTTP 래퍼
' 모든 AI 함수가 호출하는 공통 헬퍼 Private Function OllamaCall(prompt As String) As String Dim http As Object Set http = CreateObject("MSXML2.XMLHTTP.6.0") http.Open "POST", _ "http://localhost:11434/api/generate", False http.setRequestHeader "Content-Type", "application/json" Dim body As String body = "{""model"":""llama3.2"",""prompt"":""" & _ EscapeJSON(prompt) & """,""stream"":false}" http.send body OllamaCall = ParseResponse(http.responseText) End Function

EscapeJSON 헬퍼 필수. 본문 따옴표가 JSON 깨뜨림.
