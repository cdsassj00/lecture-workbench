직전 n-1개 단어로 다음 단어 확률
단순 통계 모델. 긴 문맥 불가 · 의미 파악 약함. 검색엔진 · 맞춤법 검사에 쓰였다. 공공 문서 해석에는 한계가 명확했다.
  • 확률표 기반 · 학습 불필요
  • 문맥 길이 3~5 단어가 한계
  • 한국어 조사·어미 처리 부실
STATISTICAL · N-GRAM
세대 02 · RNN · LSTM · 2014~2017
은닉 상태에 문맥을 누적
순차 처리로 문장 의미 누적. 기계번역 · 챗봇에 쓰임. 긴 문장에서 정보 소실(gradient vanishing). 병렬화 불가로 학습 속도 한계.
  • Google 번역 · Seq2Seq 기반
  • 긴 문장 끝에서 앞 내용 잊음
  • 학습 시간 · 비용 급증
SEQUENTIAL · LSTM · GRU
세대 03 · Transformer · 2017~
Self-Attention · 병렬 처리
2017년 Google "Attention is All You Need" 논문. 문장 전체를 한 번에 처리. GPT · BERT · Bard · PaLM · Claude의 공통 기반. 8회차에서 원리 체감.

Attention은 문장 속 중요 단서를 가려내는 계산
공공 문서는 조건절·예외 조항·반복성 같은 한두 토큰에서 해석이 뒤집힌다. Attention 가중치가 어디에 쏠리는지 시각화로 확인한다.
EXAMPLE · 실제 민원검게 강조된 토큰이 답변 품질을 결정한다
"버스 노선 증설 관련하여 주민 300명 서명 받아 제출합니다. 관련 민원은 2024년에도 제출했으나 반영되지 않았습니다. 이번에는 시장 답변 요청드립니다."
ATTENTION 약한 모델 · 초안
"주민 300명이 버스 노선 증설을 요청했습니다. 관계 부서와 협의하여 검토하겠습니다."
반복 민원임을 모르고, 시장 답변 요청도 누락 → 담당자가 처음부터 다시 작성.
ATTENTION 강한 모델 · 초안
"2024년에 이어 재차 제출되신 민원으로, 당시 미반영 사유는 …이며, 요청하신 시장 명의 답변은 별도 결재 라인에 올렸습니다."
반복성 · 미반영 이력 · 수신자 지정을 모두 포착 → 담당자는 사실 확인만.

Q·K·V 세 가지 벡터로 Attention이 계산된다
Q(Query·질문) · K(Key·단서 주소) · V(Value·실제 내용). 수식은 외우지 않는다. 세 벡터의 역할만 감각으로 잡고 멀티헤드로 넘어간다.
Q · QUERY
지금 알고 싶은 질문
현재 토큰이 다른 문장 속에서 어떤 정보를 찾고 있는지에 해당한다. 민원 답변을 생성할 때는 "이 민원의 핵심 요구가 무엇인가"라는 질문이 Q의 역할을 한다.
좋은 프롬프트가 좋은 답변을 만드는 이유 — 프롬프트가 모델의 Q를 명확히 설정하기 때문이다.
K · KEY
문장 속 단서의 주소
다른 토큰들이 각자 무엇을 담고 있는지 표시하는 표식이다. 법령 조문 해석에서는 조건 · 예외 · 적용 범위를 담은 단어들이 강한 K가 되고, Q(질문)와 매칭될 때 활성화된다.
검색 엔진의 키워드 매칭과 유사한 원리다. Q와 K가 잘 맞을수록 다음 단계(V)가 정확해진다.
V · VALUE
실제로 가져올 내용
중요한 단서로 판단된 토큰의 정보를 얼마나 섞어 답변에 반영할지 정한다. 최종적으로 답변의 근거 밀도와 정확도를 좌우하는 값이다.
문맥을 잘못 읽으면 잘못된 근거가 "자연스럽게" 생성되는 이유 — V가 엉뚱한 K를 반영했기 때문이다.
